Codex Skill の対話モードを使って、設計のセカンドオピニオンを取得してください。

## 手順

1. ユーザーが指定したトピックに関連するファイルを特定する
2. Codex 対話モード (`mcp__codex__codex`) でセッションを開始する:
   - 対象ファイルの現在の内容を文脈として渡す
   - 設計意図と検討中の選択肢を説明する
   - 以下を求める:
     - 設計案の評価
     - 潜在的リスクや失敗モード
     - 代替アプローチの提案
3. Codex と複数ターン対話し、設計を深掘りする
4. 結論をまとめてユーザーに報告する
