# FlEq ランディングページ デザイン仕様

## 概要

FlEq を知らない人に「何ができるか」を伝え、インストールに誘導するランディングページを GitHub Pages で公開する。

## 決定事項

| 項目 | 決定 |
|------|------|
| 目的 | ランディングページ（紹介 + インストール誘導） |
| 言語 | 日本語のみ |
| デザイン | クリーン＆ミニマル（白基調 + ライト/ダーク切替） |
| 技術スタック | 素の HTML / CSS / JS（依存ゼロ） |
| 配置 | `docs/site/` ディレクトリ |
| ホスティング | GitHub Pages（ソース: main ブランチ `docs/site/`） |

## ファイル構成

```
docs/site/
├── index.html       # メインページ
├── style.css        # スタイルシート（ライト/ダーク両対応）
├── script.js        # ダークモード切替等の軽量スクリプト
└── assets/          # スクリーンショット等の画像
```

## ページ構成（上から順）

### 1. ナビバー

- 左: FlEq ロゴ（テキスト）
- 右: セクション内リンク（機能 / 対応区分 / インストール） + GitHub リンク + ダークモードトグルボタン
- スクロール時に上部固定（sticky）

### 2. ヒーローセクション

- FlEq のロゴ（大きめテキスト）
- キャッチコピー: 「地震・津波・EEW・火山情報をリアルタイムに CLI で受信」
- npm インストールコマンド（コピー可能）: `npm install -g @sayue_ltr/fleq`
- バージョン + ライセンス表記

### 3. 機能紹介（FEATURES）

3x2 グリッドカードで 6 つの主要機能を紹介:

| 機能 | 説明 |
|------|------|
| リアルタイム受信 | WebSocket による自動再接続・複線接続 |
| 色分け表示 | 震度・レベルに応じた配色（CUD 準拠テーマ） |
| デスクトップ通知 | カテゴリ別 ON/OFF・通知音 |
| EEW 同時追跡 | 差分表記・ログ記録 |
| REPL 操作 | 実行中に設定変更・状態確認 |
| 低メモリ対応 | Raspberry Pi 等の ARM デバイスでも動作 |

### 4. スクリーンショット（SCREENSHOT）

- 背景色を変えて視覚的にセクションを区切る
- CLI の出力例をターミナル風フレームで表示
- 実際のスクリーンショット画像に差し替え予定（初期実装ではプレースホルダー）

### 5. 対応区分（SUPPORTED CATEGORIES）

2x2 グリッドで 4 カテゴリ。各カードに電文種別の詳細を記載:

| カテゴリ | 区分名 | 電文種別 |
|----------|--------|----------|
| 地震・津波関連 | `telegram.earthquake` | 震度速報 / 震源情報 / 震源・震度情報 / 長周期地震動 / 津波警報・注意報 |
| 緊急地震速報 | `eew.forecast` / `eew.warning` | EEW 予報 / EEW 警報 / 同時追跡・差分表記・ログ記録 |
| 火山関連 | `telegram.volcano` | 噴火警報 / 噴火速報 / 降灰予報 / 火山解説情報 |
| 南海トラフ | `telegram.earthquake` (VYSE*) | 南海トラフ地震関連情報（臨時・調査中・関連解説情報） |

### 6. インストール手順（GETTING STARTED）

3 ステップ形式:

1. **インストール**: `npm install -g @sayue_ltr/fleq`（または `npx` での単発実行も紹介）
2. **初期設定**: `fleq init`（対話形式で API キー・受信設定を行う）
3. **起動**: `fleq`

補足情報:
- dmdata.jp のアカウントと API キーが必要
- Node.js 18 以上が必要
- 対応 OS: macOS 10.13+ / Linux (x64/ARM) / Windows 10+
- 受信する情報の種類によっては dmdata.jp の有料契約が必要

### 7. フッター

- ライセンス表記（MIT）
- 外部リンク: GitHub リポジトリ / npm パッケージ / dmdata.jp
- 気象庁データの帰属表記:「地震情報・津波情報等は気象庁が発表したものを dmdata.jp 経由で受信しています」

## ライト/ダーク切替

- `prefers-color-scheme` メディアクエリで OS 設定に自動追従
- ナビバーのトグルボタンで手動切替可能
- 選択は `localStorage` に保存し、次回訪問時に復元
- CSS カスタムプロパティ（CSS 変数）で配色を管理し、`data-theme` 属性で切替

## レスポンシブ対応

- モバイルファースト設計
- 機能カード: モバイルで 1 列、タブレットで 2 列、デスクトップで 3 列
- 対応区分: モバイルで 1 列、デスクトップで 2 列
- ナビバー: モバイルではフォントサイズ縮小で対応（リンク数が少ないためハンバーガーメニュー不要）

## GitHub Pages 設定

GitHub Pages のブランチ設定では `/` か `/docs` のみ選択可能で、サブフォルダ (`/docs/site`) は直接指定できない。
そのため GitHub Actions ワークフローでデプロイする:

- Source: `GitHub Actions`
- ワークフロー: `.github/workflows/pages.yml`
- トリガー: `main` ブランチの `docs/site/**` への push
- 処理: `docs/site/` の内容を `actions/upload-pages-artifact` + `actions/deploy-pages` でデプロイ

### スクリーンショットのプレースホルダー

初期実装ではスクリーンショット画像がないため、ターミナル風にスタイリングした `<div>` で擬似的な出力例を表示する。実際の画像が用意でき次第差し替える。
