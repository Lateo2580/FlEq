# 電文タイプ別 表示リファレンス

各電文タイプを受電した際のターミナル表示フォーマットの一覧です。
表示処理は以下の4ファイルに分割されています:

- `src/ui/formatter.ts` — 共通表示（フレーム描画・RenderBuffer・recap・表示モード制御）
- `src/ui/eew-formatter.ts` — 緊急地震速報（EEW）の表示
- `src/ui/earthquake-formatter.ts` — 地震・津波・テキスト系・南海トラフ・長周期地震動の表示
- `src/ui/volcano-formatter.ts` — 火山情報の表示

## フレームシステム

すべての電文は「フレーム」で囲まれて表示されます。フレームのスタイルは情報の重大度（`FrameLevel`）に応じて切り替わります。

### フレームレベル一覧

| レベル | 罫線文字 | 色 (CUD) | RGB | HEX | 用途 |
|--------|----------|----------|-----|-----|------|
| `critical` | `╔ ╗ ╚ ╝ ═ ║ ╠ ╣` (二重線) | 朱赤 (バーミリオン) | `rgb(213, 94, 0)` | `#D55E00` | 震度6弱以上、EEW警報、大津波警報、南海トラフ巨大地震警戒 |
| `warning` | `╔ ╗ ╚ ╝ ═ ║ ╠ ╣` (二重線) | オレンジ | `rgb(230, 159, 0)` | `#E69F00` | 震度4以上、EEW予報、津波警報、南海トラフ注意/調査中 |
| `normal` | `┌ ┐ └ ┘ ─ │ ├ ┤` (単線) | スカイブルー | `rgb(86, 180, 233)` | `#56B4E9` | 震度3以下の地震情報、長周期階級2 |
| `info` | `┌ ┐ └ ┘ ─ │ ├ ┤` (単線) | グレー | `rgb(132, 145, 158)` | `#84919E` | テキスト情報、調査終了、長周期階級1以下 |
| `cancel` | `┌ ┐ └ ┘ ─ │ ├ ┤` (単線) | ラズベリー | `rgb(204, 121, 167)` | `#CC79A7` | 取消報 |

### フレーム構造

```
╔══════════════════════════════════════════════════════════╗   ← frameTop
║ (コンテンツ行)                                          ║   ← frameLine
╠══════════════════════════════════════════════════════════╣   ← frameDivider
║ (コンテンツ行)                                          ║   ← frameLine
╚══════════════════════════════════════════════════════════╝   ← frameBottom
```

デフォルト幅: 60 文字（`config.tableWidth` で変更可能）

### 表示モード

`config.displayMode` または REPL の `mode` コマンドで切り替え可能。

| モード | 説明 |
|--------|------|
| `normal` | フルフレーム表示 (デフォルト) |
| `compact` | 1行サマリー表示。長時間モニタリング向け |

#### コンパクトモードの表示例

```
[緊急] EEW警報  #1  宮崎県南部  震度6弱  M6.8
[警告] EEW予報  #10  豊後水道  震度5弱  M5.8
[情報] 震源・震度に関する情報  千葉県東方沖  M4.2  震度3  津波なし
[警告] 津波警報・注意報・予報  津波警報・津波注意報  岩手県, 宮城県, 青森県太平洋沿岸
[取消] EEW取消  #1  豊後水道
```

### アクセシビリティ: 重大度テキストラベル

色が見えない環境でも重大度を識別できるよう、タイトル行末尾に以下のテキストラベルが付与されます。

| レベル | ラベル |
|--------|--------|
| `critical` | `[緊急]` |
| `warning` | `[警告]` |
| `normal` | `[情報]` |
| `info` | `[通知]` |
| `cancel` | `[取消]` |

## 地震情報 (VXSE51/52/53/61)

**関数**: `displayEarthquakeInfo(info: ParsedEarthquakeInfo)`
**パーサ**: `parseEarthquakeTelegram()`

### 電文タイプ

| コード | 名称 |
|--------|------|
| VXSE51 | 震度速報 |
| VXSE52 | 震源に関する情報 |
| VXSE53 | 震源・震度に関する情報 |
| VXSE61 | 顕著な地震の震源要素更新のお知らせ |

### フレームレベル判定

| 条件 | レベル |
|------|--------|
| `infoType` = 取消 | `cancel` (ラズベリー) |
| 最大震度 ≥ 6弱 | `critical` (バーミリオン) |
| 最大震度 ≥ 4 | `warning` (オレンジ) |
| その他 | `normal` (スカイブルー) |

| 「警報」含む | `津波警報` | バーミリオン |

### 表示構成 (上から順)

| セクション | 条件 | 内容 |
|-----------|------|------|
| バナー | 常に表示 | 3行の背景色バナー。取消=ラズベリー, 警報=赤系, 予報=黄系。`colorIndex` で色分け。震源地名をバナーテキストに含む |
| テスト電文バッジ | `isTest = true` | ` テスト電文 ` (ラズベリー背景) |
| PLUM法ラベル | `maxIntChangeReason = 9` | `PLUM法 による予測震度変化` (ラズベリー) |
| ── 区切り ── | | |
| カード行 | 取消以外 | `{infoType}  │  最大予測震度 {N}  │  長周期階級 {N}  │  M{x.x}  │  深さ {N}km`。`activeCount ≥ 2` 時は infoType の前に `同時{N}件発生中` (オレンジ) を付記。`diff.previousMaxInt` がある場合は `最大予測震度 {旧} → {新}` に変化 |
| ── 区切り ── | | |
| ヘッドライン | `headline` がある場合 | 太字白 |
| ── 区切り ── | | |
| 仮定震源ラベル | `isAssumedHypocenter` | `仮定震源要素 (震源未確定・PLUM法による推定)` (ラズベリー) |
| 震源詳細 | `earthquake` がある場合 | 震源地名（変更時は `(変更)` 付記, スカイブルー） / 発生時刻 / 規模（差分あり: `M旧 → M新`） / 深さ（差分あり: `旧 → 新`） |
| 取消メッセージ | 取消時 | `この地震についての緊急地震速報は取り消されました。` (ラズベリー) |
| ── 区切り ── | | |
| 予測震度一覧 | 非取消時 | `震度{N}: {地域名}` + オプションタグ |
| 到達地域一覧 | 到達地域あり | `既に主要動到達と推測:` (バーミリオン) + 地域名リスト |
| ── 区切り ── | | |
| 最終報テキスト | `nextAdvisory` あり | スカイブルー |
| EventID | `eventId` あり | グレー |
| ── 区切り ── | | |
| フッター | 常に表示 | `{電文コード}  {発表時刻}  {発表官署}` (グレー) |

### 地域名に付くタグ

| タグ | 色 | 条件 |
|------|----|------|
| `[PLUM]` | ラズベリー | `area.isPlum = true` |
| `[到達]` | バーミリオン | `area.hasArrived = true` |
| `[長周期N]` | 階級色 | `area.lgIntensity` が 1 以上 |

### 仮定震源要素時の制限

`isAssumedHypocenter = true` の場合、以下は非表示:
- カード行のマグニチュード・深さ
- 震源詳細の規模・深さ

### 表示例 (大津波警報 / critical / ワイドテーブル: 幅80以上)

## カラーテーマカスタマイズ

表示色は `theme.json` ファイルを編集することでカスタマイズできます。テーマ管理は `src/ui/theme.ts` で定義されています。

### theme.json の配置場所

Configファイルと同じディレクトリに配置します:

- **macOS**: `~/Library/Application Support/fleq/theme.json`
- **Linux**: `~/.config/fleq/theme.json`
- **Windows**: `%APPDATA%\fleq\theme.json`

`XDG_CONFIG_HOME` が設定されている場合はそちらが優先されます。

### theme.json の構造

```json
{
  "palette": {
    "gray": "#84919E",
    "sky": "#56B4E9",
    "blue": "#0072B2",
    "blueGreen": "#009E73",
    "yellow": "#F0E442",
    "orange": "#E69F00",
    "vermillion": "#D55E00",
    "raspberry": "#CC79A7",
    "darkRed": "#7A1E00"
  },
  "roles": {
    "frameCritical": "vermillion",
    "intensity7": { "bg": "darkRed", "fg": "#FFFFFF", "bold": true },
    "hypocenter": { "fg": "yellow", "bold": true }
  }
}
```

- **palette**: 9色のベースカラーを HEX (`#RRGGBB`) で定義。変更したい色だけ記述すればOK
- **roles**: 各表示要素のスタイルを定義。文字列ならパレット名 or HEX を前景色として使用。オブジェクトなら `fg`/`bg`/`bold` を個別指定可能

### ロールスタイルの指定方法

| 形式 | 例 | 説明 |
|------|-----|------|
| パレット名 (文字列) | `"sky"` | パレットの色を前景色として使用 |
| HEX (文字列) | `"#FF0000"` | HEX値を前景色として使用 |
| オブジェクト | `{ "fg": "sky", "bg": "#000", "bold": true }` | fg/bg/bold を個別指定 |

`fg` と `bg` にはパレット名または HEX 値を指定できます。

### REPL `theme` コマンド

| コマンド | 動作 |
|----------|------|
| `theme` | テーマ概要（パレットスウォッチ・パス・カスタマイズ有無）を表示 |
| `theme path` | theme.json の絶対パスを表示 |
| `theme show` | 全パレット色・全ロールスタイルを一覧表示 |
| `theme reset` | デフォルト theme.json を書き出す (確認あり) |
| `theme reload` | theme.json を再読込して反映する |
| `theme validate` | theme.json を検証して問題点を表示 |

### カスタマイズ手順

1. `theme reset` でデフォルトの theme.json を生成
2. theme.json をエディタで編集（palette の HEX値や roles のスタイルを変更）
3. `theme reload` で変更を反映
4. `colors` コマンドで変更後の色を確認

theme.json を削除すると自動的にデフォルトテーマにフォールバックします。
